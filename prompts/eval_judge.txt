## Scoring Dimensions

Rate each dimension from 1 (poor) to 5 (excellent):

### 1. val_bpb Accuracy
- 5: Extracted value exactly matches a bpb number in the source text
- 4: Extracted value is close but has minor rounding differences
- 3: A bpb value exists in the source but the wrong one was extracted
- 2: A bpb value exists in the source but was missed (null in output)
- 1: Completely wrong value extracted, or value fabricated when none exists
- If no bpb is mentioned and output is null, score 5

### 2. Technique Completeness
- 5: All techniques mentioned in the source are captured
- 4: One minor technique missed
- 3: 2-3 techniques missed
- 2: Several important techniques missed
- 1: Most techniques missed

### 3. Technique Precision
- 5: All extracted techniques are genuinely present in the source
- 4: One questionable entry
- 3: 2-3 hallucinated or misinterpreted techniques
- 2: Several hallucinated techniques
- 1: Most techniques are not in the source

### 4. Method Naming Consistency
- 5: All method names use canonical, consistent naming (e.g., "OrthoInit" not "Orthogonal initialization", "EMA" not "exponential moving average")
- 4: One inconsistent name
- 3: Several ad-hoc or verbose names that should be standardized
- 2: Pervasive inconsistency
- 1: Names are mostly free-form descriptions, not standardized
- Refer to the KNOWN METHOD NAME VARIANTS in the audit context above for canonical forms

### 5. Architecture Appropriateness
- 5: Architecture field is a clean base type (e.g., "Transformer", "Mamba", "LSTM")
- 4: Mostly clean but includes one extra detail
- 3: Contains moderate verbosity (e.g., "12L/768d Transformer")
- 2: Very verbose, includes full config
- 1: Architecture field is a technique dump, not an architecture name
- If architecture is null and none is mentioned, score 5

### 6. Category Correctness
- 5: All techniques are in the correct category
- 4: One minor miscategorization
- 3: 2-3 techniques in wrong categories
- 2: Several miscategorizations
- 1: Systematic miscategorization
- Watch for: techniques dumped into "other" when they belong in an existing category (see OVERUSE OF 'other' in audit context), and cross-category overlaps

### 7. Vocabulary Currency
- 5: All technique names reflect current competition terminology — newer methods are recognized and correctly named
- 4: One emerging technique missed or given a non-standard name
- 3: Several newer techniques (e.g., Gated Attention, Value Residual, Parallel Muon, GPTQ-lite, Tight SWA) are missed, miscategorized, or given ad-hoc names
- 2: Most newer techniques are poorly handled
- 1: The extraction appears stuck on early-competition vocabulary, failing to capture technique evolution
- Refer to EMERGING TECHNIQUES in the audit context above
- For early-period PRs that don't use newer techniques, score 5
